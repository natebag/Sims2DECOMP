概要

このソフトウェアのインストレーションおよびその後の利用については、評価ライセンス(ProDGngc_EL.txt)かあるいはスタンダード・ユーザ・ライセンス(ProDGngc_SUL.txt)に従うものとします。

ソフトウェアはひとつのInstallshieldファイルからインストールされ、Dolphin SDKディレクトリにあるファイルは一切変更しません。

インストーラを起動すると、インストレーションパスを選択した後、Visual Studio Integrationアドインが必要かどうか聞いてきます。インストーラは環境変数とレジストリを設定します。

注意：Visual Studio Integrationアドインが必要な場合には、Visual Studioが予めインストールされ最低でも1度は起動されていなければなりません (当社が変更する前にレジストリを変えておくため)。

SDK 専用のパッチ

Nintendo SDKの最新のバージョンである2003年5月8日版にはパッチは不要です。

ファイル'NintendoSDKPatches.zip' にはNintendo SDKの複数バージョンに対する変更が含まれています。

ProDG をインストールした後、このzipファイルの内容を、インストールしたProDG と同じディレクトリに解凍します。ディレクトリ階層を変えないように注意してください。ここで、Nintendo SDK の各バージョンに対応したzipファイルを含むディレクトリがいくつも作られます。

.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0_Patch4.zip
.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0.zip
.\Nintendo Headers for SDK Sep 8 2001\DolphinSDK1.0.zip
.\Nintendo Makefiles\SDK_2001_Sep_8_SNv3.zip
.\Nintendo Makefiles\CP_2001_April_18_SNv4.zip
.\Nintendo Makefiles\MusyX_2001_June_22_SNv1.zip

ご自分のシステムに必要なのはどのパッチかを判別し、それを指示に沿って当ててください(パッチには説明が付属しています)。

.2001年12月12日付SDK \DolphinSDK1.0_Patch4.zip用の\Nintendoヘッダ

2001年12月12日付のSDKおよび同日のSDKパッチ4を使用している場合には、ファイルに付属している説明に従ってください。

このパッチは、DolphinSDK1.0.zip パッチに追加して適用してください。

.2001年12月12日付SDK \DolphinSDK1.0.zip用の\Nintendoヘッダ

2001年12月12日付のSDKを使用している場合には、ファイルに付属している説明に従ってください。

.2001年9月8日付SDK \DolphinSDK1.0.zip用の\Nintendoヘッダ

9月8日付の SDKを使用している場合には、ファイルに付属している説明に従ってください。

makefile 更新する必要があります (下記参照)。

.\Nintendo Makefile \SDK_2001_Sep_8_SNv3.zip

9月8日付SDK makefile の変更

9月8日付のSDKを使用している場合には、内容を $(DOLPHIN_ROOT)\build\buildtools に解凍してください。

.\Nintendo Makefile \CP_2001_April_18_SNv4.zip

CP SDKをご利用中の場合は、 内容を $(CP_ROOT)\build\buildtoolsplease に解凍してください。

.\Nintendo Makefile \MusyX_2001_June_22_SNv1.zip

2001年6月22日版のSDKをご利用中の場合は、内容を $(MUSYX_ROOT)\build\buildtools に解凍してください。

STLのサポート

STLを使用したい場合には、.\STL\ReadMeSTL.txtファイルにある手順に従ってください。

2001年４月18日版キャラクタパイプラインデモ

パッチについてはこのページのインストールセクションに記載されています。

任天堂はキャラクター・パイプラインのデモを別個CDで供給しています。これらのデモは以下のディレクトリc:/cp/build/demos/demo_name を用いてビルド可能で、問題なく動作します。

ビルドする前に環境変数PLATFORM=HW2 （DOLPHIN_PLATFORM=HW2変数に追加という形で）がCPデモに依存していることを確認してください。 また、二つの環境変数が正しくセットされていることを確認してください。cygwin なしでビルドしたい場合は、以下のようにDOSシンタックスの変更が必要かもしれません。

set CP_ROOT=C:/cp
set CP_X86_BIN=C:\cp\x86\bin

//c/cpスタイルの環境変数を保持したい場合は、commondefsおよびmodulerulesを絶対ではなく相対にするよう、個別の「makefiles」を以下のように修正してください。

include $(CP_ROOT)/build/buildtools/commondefs
include ../../../build/buildtools/commondefs

"\cp\build\libraries\shader\src\shaderExecute.c"のソフトウェアにはバグがあります。

"shaderExecute.c"の740行目は次のように読んでください。

for ( i = 0; i < 4; i ++ )

次のようには読まないでください。

for ( i = 0; i <= 4; i ++ )
