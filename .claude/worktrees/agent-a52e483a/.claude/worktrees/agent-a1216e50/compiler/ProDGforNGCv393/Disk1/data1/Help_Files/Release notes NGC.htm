  • Add: GX Display List Pane

  • Change: Floating point comparison x==y will now return false if either x or y is a NaN

Target Manager

  • Add: "Help->Contents" menu item to display ngctm.chm file
  • Removed: dependency on LanguageTM.dll file. Resources are now built into the executable
  • Change: Renamed the 'Auto stop at main' check box to 'Stop execution at main'

Compiler

  • Add: Support for force link once switch
  • Add: Support for -fshort-wchar in the C++ compiler
  • Add: -Wpromote-doubles, generate warnings if float types get promoted to doubles by the compiler (most commonly as parameters)
  • Add: -fdefault-single-float, treat all real constants as float type even if missing the 'f' type postfix
  • Add: no alias attribute for function prototypes
  • Fix: pre-inc/dec not using regnames
  • Fix: output of DWARF data for nested templates
  • Fix: inlined functions getting bad high PC offset in DWARF data
  • Fix: spurious warning in C when passing non-const type to const param
  • Fix: missing .size on boundless static local arrays
  • Fix: DWARF low PC using a weak label. Now uses local label

Linker

  • Add: Support for -strip-unused-data, removes unused function code and data
  • Add: Support for -sn-full-map, provides additional information in the map file
  • Add: Support for -sn-no-dtors, marks destructors as unused

General

Installation and use of this software is governed by the Evaluation License (ProDGngc_EL.txt) or the Standard User License (ProDGngc_SUL.txt).

The software is installed from a single Installshield file. The installer does not alter files in the Dolphin SDK directories.

Run the installer - it gives you the option of selecting the installation path and whether you want the Visual Studio Integration add-in. It sets up environment variables and the registry.

Note: If you want the Visual Studio Integration add-in then you must have already installed Visual Studio and run it at least once (alters registry which needs to be done before we alter it).

SDK Specific Patches

There are NO patches required for the most recent version of the Nintendo SDK, dated 8th May 2003.

The file 'NintendoSDKPatches.zip' contains changes to some versions of the Nintendo SDK.

After installation of ProDG, extract the contents of this zip to the directory in which ProDG is installed, making sure to preserve the directory structure. This will create a number of directories, containing zip files specific to different versions of the Nintendo SDKs:

.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0_Patch4.zip
.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0.zip
.\Nintendo Headers for SDK Sep 8 2001\DolphinSDK1.0.zip
.\Nintendo Makefiles\SDK_2001_Sep_8_SNv3.zip
.\Nintendo Makefiles\CP_2001_April_18_SNv4.zip
.\Nintendo Makefiles\MusyX_(all versions)_SNv1.zip

Determine which patches are relevant to your installation and apply them as instructed (each patch is accompanied by instructions).

.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0_Patch4.zip

If you are using the Dec 12th 2001 SDK, and the Dec 12th 2001 SDK Patch 4, please follow the instructions accompanying this file.

This patch should be applied in addition to the DolphinSDK1.0.zip patch.

.\Nintendo Headers for SDK Dec 12 2001\DolphinSDK1.0.zip

If you are using the Dec 12th 2001 SDK please follow the instructions accompanying this file.

.\Nintendo Headers for SDK Sep 8 2001\DolphinSDK1.0.zip

If you are using the Sep 8th 2001 SDK please follow the instructions accompanying this file.

You will also need to update the makefiles (see below).

.\Nintendo Makefiles\SDK_2001_Sep_8_SNv3.zip

Changes to Sep 8th SDK makefiles.

If you are using the Sep 8th 2001 SDK please unzip the contents to $(DOLPHIN_ROOT)\build\buildtools.

.\Nintendo Makefiles\CP_2001_April_18_SNv4.zip

If you are using the CP SDK please unzip the contents to $(CP_ROOT)\build\buildtools.

.\Nintendo Makefiles\MusyX_(all versions)_SNv1.zip

If you are using any version of the MusyX SDK, up to and including v 2.0, please unzip the contents to $(MUSYX_ROOT)\build\buildtools.

STL support

If you wish to use STL, follow the instructions in file .\STL\ReadMeSTL.txt

Character Pipeline 18 April 2001 SDK

Required patches are covered in the SDK Specific Patches section of the document.

Nintendo supplies the Character Pipeline Demos on a separate CD. Those demos can be built using the makefile in each of the c:/cp/build/demos/demo_name directories and should work just fine.

Before building these make sure you have the environment variable PLATFORM=HW2 as the CP demos depend on it (this is in addition to the DOLPHIN_PLATFORM=HW2 variable). You also need to make sure the two environment variables are set correctly and you may need to change them to DOS syntax if you wish to build without cygwin, i.e.:

set CP_ROOT=C:/cp
set CP_X86_BIN=C:\cp\x86\bin

If you wish to stick with //c/cp style of environment variable please modify the individual makefiles so that the reference to commondefs and modulerules is relative and not absolute, i.e.:

include $(CP_ROOT)/build/buildtools/commondefs
include ../../../build/buildtools/commondefs

There is a software bug in "\cp\build\libraries\shader\src\shaderExecute.c"

Line 740 of "shaderExecute.c" should read

for ( i = 0; i < 4; i ++ )

and not

for ( i = 0; i <= 4; i ++ )

Further instructions regarding source editing of CP demos can be found in the readme file accompanying the patch CP_2001_April_18_SNv4.zip.
